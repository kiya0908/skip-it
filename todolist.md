# Header Component Modification and Search Implementation Plan

本计划结合最新需求和最佳实践，指导网站头部导航与游戏搜索功能的开发。

## Step 1: 修改 Header 组件（`app/components/Header.tsx`）

- **目标：** 重构网站头部，提升导航与搜索体验。
- **任务：**
    1. [x] 移除现有菜单项（如"Play Game"、"How to Play"等）。 <!-- 完成@Cursor 2024-06-10 -->
    2. [x] 添加主导航按钮： <!-- 完成@Cursor 2025-07-07 -->
        - **标签：** "Games"
        - **跳转：** `/games` 页面，展示所有游戏。
    3. [x] 添加热门游戏快捷按钮： <!-- 完成@Cursor 2025-07-07 -->
        - 热门游戏来源于 `games.json` 中 `isHeader: true` 字段（如无则不显示）。
        - 每个按钮跳转到对应游戏详情页（如 `/games/basketbros`）。
    4. [x] 集成搜索表单： <!-- 完成@Cursor 2025-07-07 -->
        - 包含文本输入框和"Search"按钮。
        - 支持回车和点击按钮触发搜索。
        - 搜索后跳转到 `/search?q=关键词`。
    5. 响应式布局，移动端热门游戏按钮可横向滚动。 <!-- 完成@Cursor 2025-07-07 -->
    6. 所有按钮和输入框应有 aria-label，提升可访问性。 <!-- 完成@Cursor 2025-07-07 -->
    7. 结构建议拆分为：主导航区、热门游戏区、搜索区，便于维护。 <!-- 完成@Cursor 2025-07-07 -->

## Step 2: 创建搜索结果页面

- **目标：** 实现游戏搜索与结果展示。
- **任务：**
    1. 新建路由及文件：`app/search/page.tsx`。
    2. 页面逻辑：
        - 从 URL 查询参数获取搜索词。
        - 加载 `data/games.json`，根据搜索词（不区分大小写）过滤游戏。
        - 支持模糊匹配和关键词高亮。
    3. 结果渲染：
        - **有结果：** 显示"Results for '[query]'，列表展示游戏图片、名称、跳转链接。
        - **无结果：** 显示"No results for '[query]'，并推荐热门游戏（`isHot: true`）。
        - 提供"返回首页"按钮。

## Step 3: 代码结构与体验优化

- **目标：** 保证代码可维护性与用户体验。
- **任务：**
    1. Header 组件结构拆分，便于后续维护。
    2. 搜索结果页封装为独立组件，便于复用。
    3. 热门游戏、header快捷游戏均数据驱动，减少硬编码。
    4. 所有图片 alt 属性优化为包含关键词。
    5. 按钮 hover 效果与移动端适配。
    6. 全站测试导航、搜索、响应式、无障碍等。

---

如需后续扩展，可考虑：
- 支持搜索建议（自动补全）。
- 热门游戏动态排序（如按访问量/评分）。
- 搜索结果分页或懒加载。
